
# 아파트 매매·전월세 실거래 데이터 분석 계획서

작성일: 2026-03-27  
대상 데이터: 아파트 매매 및 전월세 실거래 5년치 + 단지 기본정보 + 월별 거시변수(M2, 원/달러 환율, 금값, 미국/한국 기준금리)

---

## 0. 한눈에 보는 결론

현재 보유한 데이터만으로도 아래 분석은 충분히 시작할 수 있습니다.

1. **시장 스냅샷**: 월별 거래량, 중위 ㎡당 가격, 면적대별 거래 구조 변화  
2. **가격지수**: 단순 ㎡당 가격지수, 동일단지·유사면적 반복거래지수, 품질조정(hedonic) 지수  
3. **전월세-매매 연결**: 전세가율, 월세환산수익률, 갭투자 프록시, 전월세가 매매를 선행하는지 여부  
4. **단지 프리미엄 분석**: 용적률, 건폐율, 대지면적, 세대수, 주차대수 등이 가격/유동성에 주는 영향  
5. **거시연동 분석**: 금리, 환율, 금값, M2 변화가 가격/거래량/전세가율에 미치는 시차 효과  
6. **예측/경보모형**: 1~3개월 ahead 가격·거래량 예측, 과열/냉각·유동성 경보  
7. **확장 분석**: 교통, 학군, 정책, 공급물량, 소득, 환경 데이터를 붙이면 설명력과 해석력이 크게 올라감

핵심은 **거래 행(row) 단위 데이터를 월별 패널(panel)** 로 바꿔서 보는 것입니다.  
즉, `거래 단위 → 단지×면적구간×월 → 지역×월` 구조로 요약한 뒤, 거시변수와 병합하면 대부분의 분석이 가능합니다.

## 1. 분석 전에 먼저 정리할 데이터 구조

### 1.1 권장 테이블 구조

- `sale_df`: 매매 실거래
- `rent_df`: 전세/월세 실거래
- `complex_df`: 단지 기본정보(용적률, 건폐율, 대지면적, 세대수, 주차대수)
- `macro_df`: 월별 M2, 환율, 금값, 한/미 기준금리
- `panel_df`: 단지×면적구간×월 단위로 집계한 핵심 분석 테이블

### 1.2 필수 파생변수

- `month`: 거래월
- `year`, `quarter`
- `unit_price_m2 = 거래대금 / 전용면적`
- `log_unit_price = log(unit_price_m2)`
- `area_bucket`: 면적 구간(예: 60 미만, 60~85, 85~102, 102 이상)
- `parking_per_household = 주차대수 / 세대수`
- `site_area_per_household = 대지면적 / 세대수`
- `household_density = 세대수 / 대지면적`
- `turnover_rate = 월 거래건수 / 세대수`
- `price_yoy`, `price_mom`, `volume_yoy`, `volume_mom`
- `macro_lag_1`, `macro_lag_3`, `macro_lag_6`, `macro_lag_12`

### 1.3 꼭 확인할 전처리 포인트

- 동일 단지명 표준화
- 거래취소/정정 데이터 제외 여부
- 매매와 전월세를 같은 “거래대금” 컬럼으로 관리하고 있다면 **전세보증금 / 월세 / 보증금** 을 분리 저장할 수 있는지 확인
- 단지 단위와 거래 단위가 정확히 join 되는지 확인
- 면적 표기 오차(59.97 / 59.98 / 59.99 등) 구간화 기준 통일
- 극단치 제거 또는 winsorizing 기준 설정
- 일별 거래를 월별로 집계할 때 평균보다 **중위값(median)** 중심으로 볼지 결정

## 3. 분석 시 꼭 기억할 제약 조건

### 3.1 월별 거시변수는 표본이 작다
5년 월별 데이터면 거시 시계열은 약 60포인트 수준입니다.  
따라서 고차원 시계열 모형을 무리하게 쓰기보다 아래 원칙이 좋습니다.

- 레벨(level)보다 변화율/증감률(MoM, YoY)도 같이 보기
- 시차는 1/3/6/12개월 정도의 소수 후보만 테스트
- 변수 수가 많으면 Lasso/Ridge 같은 규제모형 사용
- out-of-sample rolling validation을 반드시 수행

### 3.2 가격 상승처럼 보이는 것이 사실은 “거래구성 변화”일 수 있다
예를 들어 대형면적 비중이 늘면 평균가격은 올라가도 시장이 진짜 오른 것은 아닐 수 있습니다.  
따라서 단순 평균보다 아래를 같이 봐야 합니다.

- 중위 ㎡당 가격
- 면적구간별 가격
- 동일단지·유사면적 기준 가격
- hedonic/반복거래 기반 품질조정 지수

### 3.3 전월세 분석은 컬럼 정의가 특히 중요하다
전세와 월세는 가격 개념이 다릅니다.  
월세가 있으면 아래 정보가 있어야 비교가 제대로 됩니다.

- 보증금
- 월세
- 계약유형(전세/준전세/월세)
- 계약기간/갱신 여부(있으면 더 좋음)

## A. 데이터 진단 & 시장 스냅샷

## B. 가격 수준과 품질조정 지수

### B-1. 단순 ㎡당 가격지수 + hedonic 품질조정 지수
- **상태**: ✅ / 🟡
- **핵심 질문**
  - 보이는 가격 변화가 품질변화(면적, 단지구성) 때문인지, 시장 자체 변화인지?
- **DataFrame 구현 아이디어**
  - 단순지수: 월별 중위 `unit_price_m2`
  - hedonic: `log(unit_price_m2)`를 종속변수로 하고 단지특성 + 면적구간 + 월 더미 회귀
  - 월 더미 계수로 품질조정 가격지수 생성
- **추천 산출물**
  - 단순지수 vs 품질조정지수 비교
  - 가격상승률 분해(구성효과 vs 순수 시장효과)
- **추가 데이터 제안**
  - 준공연도, 층, 동, 향, 브랜드, 재건축 여부
- **메모**
  - 지금 가진 단지 정보만으로도 1차 버전은 가능하지만, 준공연도/층 정보가 있으면 설명력이 크게 올라감

### B-2. 동일단지·유사면적 반복거래지수 (repeat-sales proxy)
- **상태**: 🟡
- **핵심 질문**
  - 같은 단지/유사 평형이 시간 지나며 얼마나 올랐는가?
  - 시장 전반의 체감 가격지수와 일치하는가?
- **DataFrame 구현 아이디어**
  - 같은 단지 + 같은/유사 전용면적 거래를 이어 붙여 반복거래 쌍 생성
  - 로그 가격차를 이용해 월별 지수 산출
  - 표본이 적으면 지역/면적구간별로 묶어서 계산
- **추천 산출물**
  - 반복거래기반 지수
  - 거래쌍 수 추이
  - 단순지수/hedonic지수/반복거래지수 비교
- **추가 데이터 제안**
  - **사실상 중요**: 층, 동, 세부 평형코드, 가능하면 유닛 식별키
- **메모**
  - 유닛 식별키가 없으면 “정확한 same unit”이 아니라 “유사유형 반복거래”가 됨

## C. 전월세-매매 연계

### C-1. 전세가율·월세환산수익률·rent-to-price ratio
- **상태**: 🟡
- **핵심 질문**
  - 매매 대비 전세/월세가 얼마나 붙어 있는가?
  - 어느 시기/단지/면적대에서 매매보다 임대가 더 빨리 올랐는가?
- **DataFrame 구현 아이디어**
  - `sale_panel`과 `rent_panel`을 `complex × area_bucket × month` 기준으로 매칭
  - 전세가율 = 전세보증금 / 매매가격
  - 월세환산수익률 = 연환산 임대수익 / 매매가격
  - 월세는 보증금-월세 전환 로직을 정해 통일
- **추천 산출물**
  - 전세가율 분포
  - 단지별 전세가율 heatmap
  - 월세환산수익률 지도/랭킹
- **추가 데이터 제안**
  - **필수에 가까움**: 보증금과 월세를 분리한 원자료
  - 전월세전환율 또는 환산 기준금리
  - 계약유형(전세/월세/준전세)

### C-2. 전월세가 매매를 선행하는지, 갭투자 프록시가 작동하는지
- **상태**: 🟡
- **핵심 질문**
  - 전세가율 상승이 몇 개월 뒤 매매를 선행하는가?
  - “매매가 - 전세보증금” 격차가 작은 단지가 더 민감하게 움직이는가?
- **DataFrame 구현 아이디어**
  - 전세가율/임대수익률의 lead-lag correlation
  - `gap = sale_price - jeonse_deposit`
  - low-gap/high-gap 단지군 비교
  - 금리상승기 vs 금리하락기 비교
- **추천 산출물**
  - lead-lag 상관행렬
  - gap bucket별 이후 수익률/거래량 비교
- **추가 데이터 제안**
  - 전세/매매를 같은 단지·유사면적에서 더 촘촘히 연결할 수 있는 세부 면적/층 데이터
  - 계약갱신 여부
  - 전세보증보험 가입 여부(있으면 리스크 분석 확장 가능)

## D. 단지 속성 프리미엄 분석

### D-1. 용적률·건폐율·대지면적·세대수·주차대수 프리미엄
- **상태**: ✅
- **핵심 질문**
  - 어떤 단지 속성이 높은 가격과 연결되는가?
  - 속성별 효과는 선형인가, 특정 구간에서 꺾이는가?
- **DataFrame 구현 아이디어**
  - 단지 속성 분위수/binning 후 가격 비교
  - 회귀식: `log(unit_price_m2) ~ 용적률 + 건폐율 + 세대수 + 주차/세대 + 대지면적/세대 + month FE`
  - 비선형 검증: spline, quantile bin, interaction term
- **추천 산출물**
  - 속성별 가격 프리미엄 곡선
  - 단지 속성 중요도 표
- **추가 데이터 제안**
  - 준공연도(입주연차)
  - 재건축/리모델링 가능성
  - 브랜드, 커뮤니티 시설 유무

### D-2. 단지 규모와 유동성/안정성 관계
- **상태**: ✅
- **핵심 질문**

  - `turnover_rate = 월거래건수 / 세대수`
  - 단지별 변동성(rolling std), drawdown, 회복기간 계산
  - 세대수/주차/대지지분과 변동성·거래량 관계 비교

  - 단지 유동성 점수
- **추가 데이터 제안**

## E. 거시변수 연동 분석

### E-1. M2·환율·금값·한미 기준금리의 lead-lag 스크리닝
- **상태**: ✅
- **핵심 질문**
  - 어떤 거시변수가 가격보다 거래량에 먼저 반응하는가?

- **DataFrame 구현 아이디어**
  - 월별 패널과 `macro_df` 병합
  - 변수별 `lag 0/1/3/6/12` 생성
  - 상관·교차상관·단순 회귀·Granger 후보 테스트
  - price, volume, jeonse_ratio 각각 별도 테스트
- **추천 산출물**
  - lag별 상관표
  - 변수별 민감도 히트맵
- **추가 데이터 제안**
  - CPI(실질가격 변환용)
  - 가계대출/주담대 잔액
  - 주담대 금리, 국채금리, 실업률, 소득지표

### E-2. 금리 국면별 이질성 분석 / event study
- **상태**: ✅ / 🟡
- **핵심 질문**
  - 금리상승기와 금리하락기에 어떤 단지 유형이 더 민감한가?
  - 전세가율이 높은 단지군은 금리 이벤트 뒤 더 크게 움직이는가?
- **DataFrame 구현 아이디어**
  - 금리 이벤트 달 기준 pre/post window
  - high-jeonse / low-jeonse, large-complex / small-complex 등 그룹 비교
  - 단지 고정효과 + 월 고정효과 + 이벤트 더미
- **추천 산출물**
  - 이벤트 전후 누적수익률/거래량 변화
  - 금리민감도 랭킹
- **추가 데이터 제안**
  - 정책 일정표(DSR/LTV/규제지역 변경일)
  - 가계신용, 주담대 금리, 정책대출 데이터

## 9. 해석 시 유의할 점

1. **상관과 인과를 혼동하지 않기**  
   금리와 가격이 함께 움직여도 자동으로 인과라고 보기 어렵습니다.

2. **매매와 전월세는 다른 시장**  
   같은 단위로 바로 비교하지 말고, 전세가율/월세환산수익률처럼 표준화된 지표를 사용하세요.

3. **거래량은 가격보다 먼저 식는다**  
   거래량 감소는 가격조정보다 먼저 나타나는 경우가 많으므로, 거래량을 별도 핵심지표로 둬야 합니다.

4. **월별 거시변수는 표본이 짧다**  
   복잡한 딥러닝보다 해석 가능한 간명한 모형이 실전에서 더 안정적일 수 있습니다.

## 10. 조사에 참고한 분석 프레임(짧게)

아래 자료들은 본 계획서의 분석 아이디어를 정리할 때 참고한 대표 프레임입니다.

1. **OECD, Hedonic Price Indexes for Housing**  
   - 주택가격의 품질조정 지수(hedonic)와 반복거래지수(repeat-sales) 장단점을 정리한 자료

   - 미국 FHFA가 반복거래 기반(weighted repeat-sales)으로 지수를 만드는 공식 설명  
   - 링크: https://www.fhfa.gov/data/hpi

3. **한국부동산원(REB), 공동주택 실거래가격지수**  
   - 국내 실거래가격지수가 반복매매모형을 사용한다는 공식 설명  
   - 링크: https://www.reb.or.kr/rebEng/cm/cntnts/cntntsView.do?cntntsId=1376&mi=10061

4. **OECD Housing Prices Indicator**  
   - price-to-rent, price-to-income 지표 정의 참고  
   - 링크: https://www.oecd.org/en/data/indicators/housing-prices.html

5. **한국은행, 주택가격과 실물경제의 상호작용 연구**  
   - 금리 충격이 주택가격에 미치는 방향과 한계를 설명
